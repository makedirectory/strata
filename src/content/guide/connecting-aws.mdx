# Connecting to AWS

Map a **live** AWS account onto the canvas. Strata discovers resources through the
**Cloud Control API** and renders them as a typed diagram — the same nodes and
containment you'd get from an import, but sourced from what's actually running.

## Opening the dialog

Open **Connect to AWS** any of these ways:

- **Start hub** — click **+ New** in the top bar, then the **Connect to AWS** card.
- **Data menu** — click **Data ▾** in the top bar, then **Connect to AWS…**.
- **Command palette** — press **⌘K** (macOS) or **Ctrl K**, type `connect`, and
  choose **Connect to AWS (discover live resources)**.

A modal titled **Connect to AWS** opens. Close it any time with the **✕** in its
top-right corner or the **Esc** key.

## The two tabs

At the top of the dialog is a two-button tab strip:

```
[ Live scan ] [ Paste export ]
```

- **Live scan** — Strata calls AWS for you (server-side) and returns the resources.
- **Paste export** — you run the AWS CLI yourself and paste the JSON; nothing is
  sent anywhere.

**Live scan** is selected by default. Click a tab to switch.

## Live scan tab

The Live-scan body is, top to bottom:

1. A **privacy note** (🔒) reminding you that credentials are never stored.
2. **Access key ID** — labelled **Access key ID (optional)** on a local install,
   or just **Access key ID** on a hosted install (see
   [When are credentials required?](#when-are-credentials-required) below).
   Placeholder `AKIA… / ASIA…`.
3. **Secret access key** — a password field (masked).
4. **Session token (recommended)** — a password field for temporary STS
   credentials. Leave it blank when using permanent keys.
5. **Region** — a text field, defaulting to `us-east-1`. Type any region code
   (e.g. `eu-west-1`).
6. The **resource-type selector** (described next).
7. A note about temporary read-only credentials.
8. The **Discover** button.

### Choosing resource types

Below the Region field is the resource-type picker:

- A header reading **Resource types (N selected)** — the **N** is a live count of
  how many types are currently ticked. A common starter set (VPC, Subnet,
  Instance, SecurityGroup, S3 Bucket, Lambda Function, RDS DBInstance, DynamoDB
  Table) is pre-selected when the dialog opens.
- A **Select all** / **Clear** button. It toggles: when every visible type is
  ticked it reads **Clear**, otherwise **Select all**. It acts on the _currently
  visible_ (filtered) list — if you've typed in the filter, it adds or removes
  just those matches and the label shows the count, e.g. **Select all (4)**.
- A **Filter…** text box. Type part of a service name or CloudFormation type (e.g.
  `s3`, `AWS::EC2`) to narrow the list below.
- A scrollable list of checkboxes, one per discoverable type. Each row shows the
  friendly **label** and, dimmed beside it, the **CloudFormation type name**
  (e.g. `AWS::S3::Bucket`). Tick or untick rows individually.

### Running the scan

Click **Discover**. While it runs the button reads **Scanning…** and is disabled.

The **Discover** button is **disabled** when:

- no resource types are selected (**0 selected**), or
- a scan is already running, or
- you're on a **hosted** install and haven't entered both an Access key ID and a
  Secret access key.

When the scan finishes, the dialog switches to the **review** phase (see
[Reviewing and importing](#reviewing-and-importing)). If it fails, an error banner
appears at the top of the dialog and you stay on the setup form — see
[Troubleshooting](/docs/guide/troubleshooting).

## Paste export tab

No credentials on the machine running Strata, or you'd rather not hand them over?
Run the scan yourself and paste the result. Click the **Paste export** tab, then:

1. Generate the export with the AWS CLI:

   ```bash
   aws cloudcontrol list-resources --type-name AWS::S3::Bucket > buckets.json
   ```

2. Paste the output into the large text box. Strata accepts either the raw
   `list-resources` response (`{ "TypeName": "…", "ResourceDescriptions": [ … ] }`)
   or a plain JSON array of resources.
3. Click **Parse**. (The button is disabled until the box is non-empty.)

Parsing happens entirely in your browser — nothing is sent anywhere. On success
you move to the review phase; on a parse error an error banner appears and you stay
on this tab.

## When are credentials required?

Live scans run **on the server** — the Next.js process that serves the app — at the
`/api/discover` route. Where that process gets AWS credentials depends on how Strata
is deployed:

- **Local install (default).** The credential fields are **optional**. Leave them
  blank and the scan uses the server's **ambient credential chain** — your SSO
  session, shared `~/.aws` profile, environment variables, or instance role. Fill
  them in only to scan a _different_ account than the box is configured for.
- **Hosted install** (the operator set `NEXT_PUBLIC_STRATA_HOSTED=1`). You **must**
  bring your own keys. The ambient chain is disabled so visitors can't enumerate
  the operator's account. The Access-key-ID label drops the "(optional)" suffix,
  and **Discover** stays disabled until you've entered both an access key ID and a
  secret access key. A scan submitted without credentials is rejected by the server
  with a **422** and the message _"Enter AWS credentials to run a live scan, or use
  the Paste export tab."_

Either way, supplied credentials are sent over HTTPS, used in-memory for the single
scan, then discarded — never written to disk, logged, returned to the browser, or
saved into the diagram.

## IAM permissions

The scan uses the Cloud Control API's list/describe operations. The credentials you
provide (or the server's ambient identity) need, at minimum:

- `cloudcontrol:ListResources`
- `cloudcontrol:GetResource`

For each resource type, Cloud Control in turn calls that service's own read APIs
(for example `s3:ListAllMyBuckets`, `ec2:DescribeVpcs`), so the identity also needs
read access to the services you scan. The simplest safe choice is AWS's managed
**`ReadOnlyAccess`** policy.

> **Security note.** Prefer **temporary, read-only** credentials. Run
> `aws sts get-session-token` (or assume a read-only role) and paste the resulting
> access key ID, secret, and **session token** into the three fields. Avoid pasting
> long-lived root or admin keys. Discovery is list/describe only — Strata never
> mutates your account.

## Reviewing and importing

After a successful **Discover** (or **Parse**), a **review** panel appears at the
bottom of the dialog before anything touches your canvas. It shows:

- **N resource(s) ready to import** — how many discovered resources map to a known
  service in Strata's catalog.
- **· N unmapped type(s)** (only if any) — types Strata doesn't model yet, which
  will be skipped. The exact list is shown below as _"Not yet modelled (skipped):
  …"_.
- **· scanned N type(s) in `<region>`** (live scans only) — confirms the scope.
- Any per-type **warnings** from the scan, each on its own line.

Then choose how to apply the resources with one of two buttons:

- **Merge into canvas** — add the discovered resources to your current diagram,
  keeping everything you already have.
- **Replace canvas** — swap in the discovered set. If you have unsaved changes,
  Strata first shows the **"Replace the current diagram?"** confirmation with
  **Save & continue**, **Discard & continue**, and **Cancel**.

Both buttons are disabled if there's nothing mappable to import (**0 ready to
import**). After importing, the status line reads _"Imported N discovered
resource(s) (merge|replace)."_ The discovered nodes (named **"Discovered
(`<region>`)"** as a group) can be edited, connected, and rearranged like any other
node.

## What discovery does and doesn't do

- **Read-only modeling.** Discovery only lists and describes resources — Strata
  never mutates your account.
- **Credentials never leave the server or reach the diagram.** They stay in the
  server process for the duration of a scan; they're never returned to the browser,
  written to disk, or stored in a graph. Because the model keeps only the
  configuration fields Strata recognizes, stray properties (let alone secrets)
  can't end up in a node either.
- **Relationships aren't inferred from Cloud Control.** Discovered resources land as
  nodes; wire up the connections you care about on the canvas. (Containment and
  edges from a CloudFormation/Terraform [import](/docs/guide/importing-iac) are
  richer if you have the source.)
- **Nothing is scanned outside the types and region you chose** — the review makes
  the scope explicit.
